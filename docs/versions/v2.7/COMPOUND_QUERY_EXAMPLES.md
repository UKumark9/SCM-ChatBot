# Compound Query Examples - Quick Reference

**Quick guide to asking multi-domain questions in a single query**

---

## 🎯 Basic Compound Queries

### Delay + Forecasting

```
✅ "Show delays and forecast demand"
✅ "What are the delivery delays and future demand predictions?"
✅ "Show delivery performance plus demand forecast"
✅ "Analyze delays along with demand trends"
```

**Result:** Delay analysis + demand forecast + cross-insights on supply chain risks

---

### Delay + Analytics

```
✅ "Show delays and revenue performance"
✅ "What are the delays and sales metrics?"
✅ "Analyze delivery performance and customer behavior"
✅ "Show delays plus revenue trends"
```

**Result:** Delay stats + revenue analysis + impact insights

---

### Analytics + Forecasting

```
✅ "Show revenue and forecast demand"
✅ "What are sales trends and future predictions?"
✅ "Analyze customer behavior and predict demand"
✅ "Show revenue plus demand forecast"
```

**Result:** Revenue analysis + forecast + inventory planning insights

---

## 🔥 Advanced Compound Queries

### Triple Agent (Delay + Analytics + Forecasting)

```
✅ "Show delays, revenue, and forecast demand"
✅ "Analyze delivery performance, sales, and future demand"
✅ "Comprehensive report on delays, revenue, and predictions"
✅ "Show me delays also revenue and forecast"
```

**Result:** Complete supply chain view + multi-domain insights

---

### Product-Level Compound

```
✅ "Show product delays and forecast demand for electronics"
✅ "What are electronics delays and sales performance?"
✅ "Show delays for product ABC123 plus revenue analysis"
✅ "Analyze electronics delays and predict future sales"
```

**Result:** Product-specific analysis across multiple domains

---

### Category + Multi-Domain

```
✅ "Show electronics delays and forecast category demand"
✅ "What are furniture delays and revenue for that category?"
✅ "Analyze toys category: delays and sales and forecast"
```

**Result:** Category-specific multi-domain analysis

## 🎨 Query Templates

### Performance + Prediction

```
"Show [performance metric] and forecast [target]"

Examples:
• "Show delivery delays and forecast demand"
• "Show on-time rate and forecast sales"
• "Show delay trends and forecast orders"
```

### Performance + Analytics

```
"Show [performance metric] and [analytics metric]"

Examples:
• "Show delays and revenue"
• "Show on-time rate and customer satisfaction"
• "Show delivery performance and order value"
```

### Multi-Metric + Forecast

```
"Show [metric 1], [metric 2], and forecast [target]"

Examples:
• "Show delays, revenue, and forecast demand"
• "Show performance, sales, and predict growth"
• "Show delays, customers, and forecast orders"
```

### Product-Specific Multi-Domain

```
"Show [product/category] [metric 1] and [metric 2]"

Examples:
• "Show electronics delays and forecast demand"
• "Show product ABC delays and revenue"
• "Show furniture delays and sales performance"
```

## 🚫 What NOT to Do

### ❌ Too Vague

```
❌ "Show everything"
❌ "Tell me about the business"
❌ "What's happening?"
```

**Better:**
```
✅ "Show delays, revenue, and forecast"
✅ "Comprehensive supply chain report"
```

---

### ❌ Conflicting Requests

```
❌ "Show only delays but also revenue and forecast"
```

**Better:**
```
✅ "Show delays, revenue, and forecast"
```

---

### ❌ Too Many Domains (>4)

```
❌ "Show delays, revenue, customers, products, forecast, inventory, and suppliers"
```

**Better:** Break into separate queries or use:
```
✅ "Comprehensive supply chain report"
```

---

## 💪 Best Practices

### 1. Be Specific About Domains

**Good:**
```
✅ "Show delivery delays and demand forecast"
```

**Better:**
```
✅ "Show delivery delays and forecast demand for next 30 days"
```

---

### 2. Use Clear Conjunctions

**Good:**
```
✅ "Show delays forecast" (might work)
```

**Better:**
```
✅ "Show delays and forecast"
✅ "Show delays plus forecast"
```

---

### 3. Product/Category First

**Good:**
```
✅ "Show delays and forecast for electronics"
```

**Better:**
```
✅ "Show electronics delays and forecast demand"
✅ "For electronics, show delays and forecast"
```

---

### 4. Combine Related Metrics

**Good:** Separate queries
```
"What are delays?"
"What is revenue?"
"What is forecast?"
```

**Better:** Single compound query
```
✅ "Show delays, revenue, and forecast"
```

## 🎓 Learning Path

### Beginner

Start with simple 2-domain queries:
```
1. "Show delays and forecast"
2. "Show delays and revenue"
3. "Show revenue and forecast"
```

### Intermediate

Try product-level compound queries:
```
1. "Show product delays and forecast"
2. "Show electronics delays and revenue"
3. "Show product ABC delays and sales"
```

### Advanced

Use triple-agent with specifics:
```
1. "Show delays, revenue, and forecast for electronics"
2. "Analyze product ABC: delays, sales, and demand predictions"
3. "Comprehensive analysis of furniture category performance"
```

### Expert

Leverage cross-insights:
```
1. "Show high-delay products, their revenue impact, and forecast trends"
2. "Analyze supply chain: regional delays, revenue patterns, demand shifts"
3. "Strategic assessment: delivery performance vs growing demand"
```
